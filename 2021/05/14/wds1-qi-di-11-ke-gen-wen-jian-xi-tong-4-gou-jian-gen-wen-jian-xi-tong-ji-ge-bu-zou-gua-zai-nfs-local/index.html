根据 根文件系统 2 知：
最小根文件系统的项

  1. /dev/console
  2. /dev/null
  3. init (busybox)
  4. /etc/inittab
  5. 配置文件里指定的应用程序或脚本
  6. C库

    1.2. 创建设备

    查看pc系统的两个设备信息 ls /dev/console /dev/null -l，都是字符设备，5是主设备1是次设备。

    到上节的根文件系统目录下去，mkdir dev ， cd dev，
    sudo mknod console c 5 1 创建节点console，c字符设备，主设备5次设备1
    sudo mknod null 1 3 创建节点null，c字符设备，主设备1次设备3
    结果如下：

    3. init (busybox)在 根文件系统 3 已完成

    4. /etc/inittab 配置文件

    创建inittab
    mkdir etc
    vim etc/inittab
    填入内容，意思是执行一个askfirst类型的程序 叫做 "-/bin/sh"，这个程序的stdin，stdout，stderr定位到console中，
    console::askfirst:-/bin/sh

    6. C库

    在根文件系统中创建lib文件夹，然后将gcc-3.4.5-glibc-2.3.6/arm-linux/lib目录下的.so拷贝到该目录下，-d表示保持链接文件，其他不是链接文件的直接拷贝，不加-d19M加-d7M，
    mkdir lib 首先在根文件系统中创建lib文件夹
    cd gcc-3.4.5-glibc-2.3.6/arm-linux/lib
    cp *.so* /media/D/韦东山/system/fs_root/lib/ -d 然后拷贝.so到lib下 。

注意：
1 制作的根文件系统lib库的版本必须与编译内核和编译busybox的交叉工具一致（最好使用厂家推荐的），否则启动卡在：

VFS: Mounted root (yaffs filesystem).                                           
Freeing init memory: 140K                                                       
Failed to execute /linuxrc.  Attempting defaults...                             
Kernel panic - not syncing: No init found.  Try passing init= option to kernel.

2 如果是烧写根文件系统，在复制.so时不加-d，直接拷贝不链接，否则启动不了。
我的编译交叉工具的lib库在/usr/local/arm/gcc-3.4.5-glibc-2.3.6/arm-linux/lib。

5. 那些配置文件中指定的程序或脚本

暂未指定

制作映像文件（目的是烧写）

把制作好的根文件系统烧写到开发板，需要制作根文件系统。
yaffs1映像文件针对小页nand，每页512Byte。
大页nand每页2048Byte，所以要用yaffs2。
制作映像文件的工具也是需要修改的。
给的资料已经改好，在压缩包里：yaffs_source_util_larger_small_page_nand.tar.bz2。

  1. 先解压：tar xjf yaffs_source_util_larger_small_page_nand.tar.bz2
  2. 再make，cd Development_util_ok/yaffs2/utils
    make
  3. 生成的工具拷贝到pc目录
    sudo cp mkyaffs2image /usr/local/bin/，将生成的工具mkyaffs2image拷贝到pc系统目录/usr/local/bin/
  4. 给工具加权限
    sudo chmod +x /usr/local/bin/mkyaffs2image 加上可执行权限
  5. 用生成的工具将根文件系统制作成映像文件
    到根文件系统所在目录执行：mkyaffs2image fs_root fs_root.yaffs2

上面步骤完成最小根文件系统的构建。

root/etc/ (init.d，fstab)

一些配置文件
添加ps功能：

有些命令还是不能执行如ps，需要挂载由内核提供的虚拟文件系统proc，自动收集进程信息，在console中操作，
  mkdir proc 创建目录
  mount -t proc none /proc 手动挂载
查看proc下的文件，cd proc/1，pid等于1的进程，ls fd -l，fd指向0stdout，1stdin，2stderr，

自动挂载proc文件，

  1. 回到pc，在构建的根文件系统中创建文件夹proc：mkdir proc

  2. 需要修改配置文件，回到pc，在自己制作的根文件系统下，打开inittab，vim etc/inittab，添加后如下，

    console::askfirst:-/bin/sh
::sysinit:/etc/init.d/rcS

    然后创建脚本所在文件夹mkdir etc/init.d，再创建脚本，vim etc/init.d/rcS，添加内容意思挂载proc文件，

    mount -t proc none /proc

    加上可执行权限：chmod +x etc/init.d/rcS ，这样启动之后就会自动运行这个脚本，挂在proc文件。

  3. 也可用mount -a，这个命令会根据etc/fstab文件的内容来挂在哪些文件系统，rtc/fstab的格式如下，
    以上两个文件修改后内容如下：

    root/dev，使用udev

    /dev/下存放一些驱动节点。
    udev工具能够自动创建/dev/设备节点，简化版本为mdev，使用方法在busybox-1.7.0/docs/mdev.txt，

    Here's a typical code snippet from the init script:
[1] mount -t sysfs sysfs /sys
[2] echo /bin/mdev > /proc/sys/kernel/hotplug
[3] mdev -s

Of course, a more "full" setup would entail executing this before the previous
code snippet:
[4] mount -t tmpfs mdev /dev
[5] mkdir /dev/pts
[6] mount -t devpts devpts /dev/pts

    操作：

  4. 在pc上，根文件系统新建sys文件夹，mkdir sys

  5. 在etc/fstab中修改：sysfs也是虚拟文件系统

    # device mount-point   type    options  dump  fsck  order
proc      /proc        proc    defaults  0     0
sysfs     /sys         sysfs   defaults  0     0
tmpfs     /dev         tmpfs   defaults  0     0

  6. 在etc/init.d/rcS中修改：hotplug热拔插，动态加载驱动，当插入设备时内核调用hotplug，它指向mdev，/sbin/mdev这个程序就会自动创建设备节点；mdev -s表示先把有的设备都创建出来，

    # mount -t proc none /proc
mount -a
mkdir /dev/pts
mount -t devpts devpts /dev/pts
echo /sbin/mdev > /proc/sys/kernel/hotplug
mdev -s

    挂接NFS（目的是不烧写）

    服务的安装NFS服务：sudo apt-get install nfs-kernel-server -y

先在flash上启动已有的根文件系统，然后在linux系统中挂接NFS

有两个条件：1 服务器允许该目录被挂接 2 开发板去挂接

  1. nfs服务在/etc/exports中配置， 
    # /nfs/fs_root     ：要共享的路径
# *                ：*通配，表示所有网段都可以访问
# sync             ：同步写入硬盘
# no_root_squash   ：nfs客户端共享目录使用者权限

/nfs/fs_root *(rw,sync,no_root_squash)
    重启服务：sudo /etc/init.d/nfs-kernel-server restart
  2. 在开发板上挂接
    新建挂载的目录mnt，mkdir mnt
    1 mount -t nfs -o nolock 192.168.1.103:/nfs/fs_root /mnt // 默认udp
    2 mount -t nfs -o tcp,nolock 192.168.1.103:/nfs/fs_root /mnt // tcp，可以传输大文件

    直接从根文件启动

    在开发板的uboot中设置参数bootargs
    原来参数bootargs=noinitrd root=/dev/mtdblock3 init=/linuxrc console=ttySAC0,115200
    set bootargs noinitrd root=/dev/mtdblock3 init=/linuxrc console=ttySAC0,115200
    两个条件： 1 设置服务器的ip和目录 2 自己的ip
    格式在内核文件Documentation/nfsroot.txt中，
    nfsroot=[<server-ip>:]<root-dir>[,<nfs-options>]
...
ip=<client-ip>:<server-ip>:<gw-ip>:<netmask>:<hostname>:<device>:<autoconf>

对应设置为：
set bootargs noinitrd root=/dev/nfs nfsroot=192.168.1.103:/nfs/fs_root ip=192.168.1.100:192.168.1.103:192.168.1.1:255.255.255.0::eth0:off init=/linuxrc console=ttySAC0,115200
set bootargs noinitrd root=/dev/nfs nfsroot=192.168.0.103:/nfs/fs_root ip=192.168.0.11:192.168.0.103:192.168.0.1:255.255.255.0::eth0:off init=/linuxrc console=ttySAC0,115200
