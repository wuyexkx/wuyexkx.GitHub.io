开发板相关

uboot

设置bootargs： set bootargs noinitrd root=/dev/mtdblock3 rootfstype=yaffs2 init=/linuxrc console=ttySAC0
设置bootcmd： set bootcmd nand read.jffs2 0x30007fc0 kernel;bootm 0x30007fc0
set bootcmd nand read.jffs2 0x30007FC0 0X00060000 0X00200000

ping 192.168.0.120
从uboot进入菜单选项： menu
查看nand分区别名： mtd
打印内存的内容： md.w o

重新编译uboot

  1. tar xjf u-boot-1.1.6.tar.bz2 解压
  2. cd u-boot-1.1.6/ 到目录下
  3. patch -p1 < ../u-boot-1.1.6_jz2440.patch 打补丁
  4. make 100ask24x0_config 配置make
  5. make make

    重新烧写uboot

    sudo ./oflash u-boot.bin

    串口打印小工具

    minicom： sudo minicom

tftp下载

前提是开发板能ping同服务器，服务器的目录下有需要下载的文件。
需要服务器支持tftp：
首先要安装tftp服务端：sudo apt-get install tftpd-hpa //tftpd-hpa是服务器端
安装好后配置服务器的设置：sudo gedit /etc/default/tftpd-hpa，

# /etc/default/tftpd-hpa

TFTP_USERNAME="tftp"
TFTP_DIRECTORY="/media/D/tftp"
TFTP_ADDRESS=":69"
TFTP_OPTIONS="--secure"

新建目录给权限：
chmod 777 ./tftp
重新启动tftp服务：sudo service tftpd-hpa restart

  1. 开发板uboot下：
    tftp 30007FC0 uImage
    nand erase kernel
    nand write 30007FC0 kernel
  2. 开发板下载sysfile：
    tftp 30000000

    配置SSH远程

    Ubuntu默认没有安装SSH Server，使用以下命令安装：

    sudo apt-get install openssh-server
    然后确认sshserver是否启动了
    ps -e|grep ssh 或用netstat -tlp命令
    如果只有ssh-agent那ssh-server还没有启动，
    需要/etc/init.d/ssh start，如果看到sshd那说明ssh-server已经启动了。
    如果没有则可以这样启动：
    sudo/etc/init.d/ssh start

    重启服务

    sudo service ssh restart

    linux目录下的文件意义

    /proc 是虚拟文件系统，存放进程相关的状态信息，用户与内核通信的纽带之一。
    /sys 虚拟文件系统，包含所有系统硬件的层级视图，把设备和总线组成分级文件，就是展示设备驱动模型各组件的关系。
    dev/xxx 		// 设备节点，xxx为设备节点名称
proc/devices	// 主设备号和对应名称保存的文件 1 mem
proc/PID/maps	// 一个进程的所有vma在虚拟地址空间的分布情况,每个segment用一个vm_area_struct（以vma）结构体表示
proc/iomem		// 查看kernel的text data bss内存分布，是物理地址
proc/zoneinfo	// 包含由内存frame页框组成的zone内存域组成的node的信息
proc/interupts	// 包含中断信息，控制器名称 中断名称等等
/mnt			// 挂在的文件系统
/sys/class/		// mdev自动创建设备节点的根据 是 这个内核信息 /sys/class/class_name/dev_node_name/dev

linux版本号

Linux内核版本有两种：稳定版和开发版
Linux内核版本号由3组数字组成：第一个组数字.第二组数字.第三组数字

第一个组数字：目前发布的内核主版本。
第二个组数字：偶数表示稳定版本；奇数表示开发中版本。
第三个组数字：错误修补的次数。

例： 2.6.18-128.ELsmp ,

第一个组数字: 2 , 主版本号
第二个组数字: 6 , 次版本号，表示稳定版本(因为有偶数)
第三个组数字 18 , 修订版本号 ， 表示修改的次数，头两个数字合在一齐可以描述内核系列。如稳定版的2.6.0，它是2.6版内核系列。128: 表示这个当前版本的第5次微调patch ， 而ELsmp指出了当前内核是为ELsmp特别调校的 EL : Enterprise Linux ； smp : 表示支持多处理器 ， 表示该内核版本支持多处理器
