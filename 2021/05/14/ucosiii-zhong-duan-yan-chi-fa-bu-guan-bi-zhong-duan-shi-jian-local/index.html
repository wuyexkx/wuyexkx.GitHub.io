ucosiii 中断延迟发布关闭中断时间

中断相关通用基础

关闭中段时间

在操作系统中，很多时候我们会主动进入临界段，系统不允许当前状态被中断打断，故而在临界区发生的中断会被挂起，直到退出临界段时候打开中断。此部分时间，我称其为： 关闭中断时间。

中断延迟

中断延迟可以定义为，从中断开始的时刻到中断服务例程开始执行的时刻之间的时间段。
中断延迟 = 识别中断时间 + [等待中断打开时间] + [关闭中断时间]。
进入临界段的方式可以是关中断或者锁住调度器。
用锁住调度代替关中断，大大减少了关中断的时间， 也能达到进入临界段的目的。 中断延迟就是利用这种思想， 让本该在中断中完成的事情切换到任务中完成， 而且进入临界段的方式是锁定调度器， 这样子中断就不会被屏蔽， 系统能随时响应中断。

UCOSIII

通过宏选择

UCOSIII支持中断嵌套的，即高优先级的中断可以打断低优先级的中断，在UCOSIII中使用OSIntNestingCtr来记录中断嵌套次数，最大支持250级的中断嵌套，每进入一次中断服务函数OSIntNestingCtr就会加1，当退出中断服务函数的时候OSIntNestingCtr就会减1。

在编写UCOSIII的中断服务程序的时候需要使用到两个函数OSIntEnter()和OSIntExit()。

任务级调度器为函数OSSched()；
中断级调度器为函数OSIntExit()，当退出外部中断服务函数的时候使用中断级任务调度。

中断直接发布 和 延迟发布

1 直接发布

  • （1）中断产生
  • （2）中断服务程序ISR开始执行。通常中断中包含着任务等待的事件
  • （3）若ISR 使能了低于或等于原任务优先级的任务，ISR结束时，uC/OS-III 返回原任务并从被中断处继续执行
  • （4）若ISR 使能了高于原任务优先级的任务，ISR 结束后，进入调度。uC/OS-III 切换到高优先级任务
  • （5）在直接提交方法中，uC/OS-III 必须保护好临界段（通过关中断方式）

2 延迟发布

  • （1）中断产生
  • （2）ISR 被执行
  • （3）ISR 中调用”post”函数发送信号量或消息给任务。然而，它不是直接发送给任务，而是先发送到中断队列。然后中断处理函数被就绪。这是uC/OS-III 的内部任务且具有最高优先级(优先级为0)
  • （4）ISR 的最后，uC/OS-III 会切换到中断处理任务，它将中断队列中消息发送给任务。在此，我们关闭中断防止在处理中断队列时被另外的中断程序打断。最后，该任务使能中断，锁住调度器，提交信息。信息是在任务级被提交的。这样，就是在任务级完成临界段的处理了
  • （5）当中断处理任务清空中断队列时，它就会将自己停止，重新开启调度器，调用调度器选择下一个任务运行
  • （6）如果优先级更高的任务被使能，uC/OS-III 会马上换到该任务

总结：
使用延迟发布关中断的时间会非常短（且相当固定），但是任务延迟会加长，因为UCOSIII会锁住调度器并访问临界段。

  • uC/CPU 中提供了测量最大中断时间的功能
  • uC/OS-III 的ISR 前序，ISR 后序，OSIntExit()，OSIntCtxSw()操作的时间可以被测量。
  • 调用OS_TS_GET()也可以测量“post”函数的处理时间。
  • 只有在处理定时器任务时才会锁调度器。如果定时器任务中创建的定时器不多的话，任务延迟将会很短。uC/OS-III也可以测量锁调度器时间。

    UCOSIII中断延迟 ISR添加OSIntEnter()和OSIntExit()

    相比UCOSII，UCOSIII对从中断发布消息或者信号的处理有两种模式：直接发布和延迟发布两种方式。
    通过宏OS_CFG_ISR_POST_DEFERRED_EN选择：
  • 为0时使用直接发布模式，
  • 为1的时候使用延迟发布模式。
    对于开发人员来说，使用ucosiii中断延迟发布功能，只需要将宏OS_CFG_ISR_POST_DEFERRED_EN打开，在中断服务函数（ISR，通常是在中断向量表里边固定的函数名）中添加，OSIntEnter(); 和 OSIntExit();,中间的代码跟task通信一样写，其他上面提到的中断队列等机制都是ucosiii自己完成。除了添加两行代码以外，跟普通裸机中断编写没什么差别。

中断服务函数：

// 1. --------------- 以CAN的接收中断为例 ---------------------
#if CAN2_RX0_INT_ENABLE	//使能RX0中断
//CAN2接收中断函数
void CAN2_RX0_IRQHandler(void)
{
	OS_ERR err;
	CanRxMsg RxMessage;  

#ifdef SYSTEM_SUPPORT_OS	 	
OSIntEnter();	// 添加之后ucosiii自动将xxx添加到中断队列，退出ISR后进入中断队列处理任务（优先级最高0）然后发布到指定task
#endif

	CAN_Receive(CAN2, CAN_FIFO0, &RxMessage);	

	// 发布消息到消息队列 queue */
	OSQPost ((OS_Q *)&queue,
	(void *)(&CAN2_RECV_DATA),
	(OS_MSG_SIZE )sizeof (CAN2_RECV_DATA),
	(OS_OPT )OS_OPT_POST_FIFO | OS_OPT_POST_ALL,
	(OS_ERR *)&err); 
	
	// 清除挂起中断
	CAN_ClearITPendingBit(CAN2, CAN_IT_FMP0);	

#ifdef SYSTEM_SUPPORT_OS	 	
OSIntExit(); // 添加之后ucosiii自动将xxx添加到中断队列，退出ISR后进入中断队列处理任务（优先级最高0）然后发布到指定task
#endif
}

	if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
